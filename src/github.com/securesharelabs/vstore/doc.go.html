     1  /*
     2  Package vstore implements a verifiable store for CometBFT nodes.
     3  
     4  # Motivation
     5  
     6  vStore is a Go application built using [CometBFT]. It focusses on providing:
     7  
     8    - data integrity: attaching timestamp and signature information to data ; and
     9    - data redundancy: running a vstore application on top of CometBFT nodes ; and
    10    - data availability: data is available from any supporting CometBFT nodes ; and
    11    - data security: the blockchain is *not* used to store your data!
    12  
    13  vStore is built using the [cobra] command-line utility software.
    14  
    15  By default, the main function runs the vstoreCmd from `cmd/vstore.go` which starts
    16  a vStore instance and asks the user for a password to decrypt the node identity.
    17  
    18  # Examples
    19  
    20  	vstore --home=/tmp/.vfs-home --socket=unix://vfs.sock
    21  	vstore version
    22  	vstore info --home=/tmp/.vfs-home
    23  	vstore factory --home /tmp/.vfs-home --data "Message here" --commit
    24  	vstore query --home /tmp/.vfs-home --hash TRANSACTION_HASH_HEX
    25  
    26  # Commands
    27  
    28    - `vstore`: Default vStore application startup (ABCI application server).
    29    - `vstore factory`: Create digitally signed transactions for vfs nodes.
    30    - `vstore version`: Print the version number of your vStore instance.
    31    - `vstore info`: Print the current node's vStore information (State).
    32    - `vstore query`: Query your vStore instance for transactions.
    33  
    34  [cobra]: https://github.com/spf13/cobra
    35  [CometBFT]: https://github.com/cometbft/cometbft
    36  */
    37  package main
    38
